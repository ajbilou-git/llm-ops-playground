# AI/ML Glossary

| Term | Definition |
|------|-----------|
| **Agentic AI** | Autonomous AI systems capable of reasoning, planning, and using tools to complete tasks |
| **Agentic RAG** | RAG enhanced with autonomous agents that decide which sources to query and when |
| **Chunking** | Splitting documents into smaller segments for embedding and retrieval |
| **Context Window** | Maximum number of tokens an LLM can process in a single request |
| **Drift Detection** | Detecting when model performance degrades over time due to data or behavior changes |
| **Embedding** | Numerical vector representation of text, enabling semantic similarity computation |
| **Evaluation (LLM)** | Systematically measuring LLM response quality with reproducible metrics |
| **Fine-tuning** | Retraining a pre-trained model on domain-specific data to specialize it |
| **Function Calling** | LLM capability to invoke external functions with structured parameters |
| **Guardrails** | Protection mechanisms preventing an LLM from producing dangerous or incorrect content |
| **Hallucination** | When an LLM generates plausible but factually incorrect information |
| **LLM (Large Language Model)** | Neural network trained on massive text data capable of generating human-like text |
| **LLM Observability** | Tracing, measuring, and analyzing every LLM call (latency, tokens, cost, quality) |
| **LoRA** | Low-Rank Adaptation — parameter-efficient fine-tuning technique |
| **MCP (Model Context Protocol)** | Standardized protocol enabling LLMs to interact with external tools and APIs |
| **MLOps** | Practices for deploying, monitoring, and maintaining ML models in production |
| **Orchestration** | Coordinating multiple agents or services to solve complex problems collaboratively |
| **Prompt Engineering** | Crafting precise instructions to guide LLM behavior and output quality |
| **QLoRA** | Quantized LoRA — fine-tuning with reduced memory footprint |
| **RAG (Retrieval-Augmented Generation)** | Combining knowledge base search with LLM generation for grounded answers |
| **RAGAS** | RAG Assessment framework measuring faithfulness, relevancy, recall, and precision |
| **ReAct** | Reason + Act pattern — agent alternates between reasoning and taking actions |
| **Routing** | Dynamically selecting which tool, source, or model to use based on the query |
| **Semantic Search** | Finding relevant content by meaning rather than keyword matching |
| **Token** | Base unit of text for an LLM (word, subword, or character) |
| **Tool Use** | LLM capability to call external tools (APIs, databases, functions) during generation |
| **TTFT** | Time to First Token — latency before streaming the first token of a response |
| **Vector Database** | Database optimized for storing and searching vector embeddings at scale |
| **vLLM** | High-throughput LLM serving engine with PagedAttention |
